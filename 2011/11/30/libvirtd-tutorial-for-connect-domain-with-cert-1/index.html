libvirtd使用证书创建连接简介(上)

上集讲的是如何配置证书与证书的存放

先了解一下证书的存放位置和需要注意的一些地方

位置 安装机 描述 匹配域
/etc/pki/CA/cacert.pem 服务端 and 客户端 CA的证书 n/a
/etc/pki/libvirt/private/serverkey.pem 服务端 服务端的私钥 n/a
/etc/pki/libvirt/servercert.pem 服务端 由CA颁发的服务端签名证书 服务器的hostname必须在客户端可见,并且与CommonName匹配
/etc/pki/libvirt/private/clientkey.pem 客户端 客户端私钥 n/a
/etc/pki/libvirt/clientcert.pem 客户端 由CA颁发的客户端签名证书 Distinguished Name(DN)需要在access control list里面(tls_allowed_dn_list)
  • 安装 gnults-utils
apt-get install guntls-bin
  • 创建CA, SERVER, CLIENT对应的证书与私钥

  1. 先创建三个文件,分别命名为 ca.info, server.info, client.info 分别填上以下的内容

    file: ca.info
cn = <CA的机构>
ca
cert_signing_key
    file: server.info
organization = <CA的cn>
cn = <server的hostname>
tls_www_server
encryption_key
signing_key
    file: client.info
country = <国家>
state = <省> 
locality = <市>
organization = <CA的cn>
cn = <client的hostname>
tls_www_client
encryption_key
signing_key

  2. 创建 cakey.pem, serverkey.pem, clientkey.pem

    创建 cakey.pem
certtool --generate-privkey > cakey.pem
    创建 server.pem
certtool --generate-privkey > serverkey.pem
    创建 client.pem
certtool --generate-privkey > clientkey.pem

  3. 创建 cacert.pem, servercert.pem, clientcert.pem

    创建 cacert.pem
certtool --generate-self-signed --load-privkey cakey.pem \
	 --template ca.info --outfile cacert.pem
    创建 servercert.pem
certtool --generate-certificate --load-privkey serverkey.pem \
  --load-ca-certificate cacert.pem --load-ca-privkey cakey.pem \
  --template server.info --outfile servercert.pem
    创建 client.pem
certtool --generate-certificate --load-privkey clientkey.pem \
  --load-ca-certificate cacert.pem --load-ca-privkey cakey.pem \
  --template client.info --outfile clientcert.pem
  • 将产生的私钥跟证书放到对应的主机的对应目录下面.
  • 重启libvirtd进程
    libvirtd -l -d

配置就这样完成了,如何连接我将在下一节中描述.
